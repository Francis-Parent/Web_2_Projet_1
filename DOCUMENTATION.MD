# Système de nomenclature CSS:
J'ai choisi mon système de nomenclature parce qu'il est simple pour moi à comprendre et me fait être plus éfficace pour effectuer mes tâches.
# Variables CSS et design tokens:
Ma logique dans le nommge des tokens pour chaque variable est que je prends la variable et je le nomme par quoi ça fonction est dans le HTML comme par exemple: (header__nav), il est appelé ceci pour montrer qu'il est dans la section header et sa fonction est de faire la navigation.
# Liste des composants réutilisables:
Ma liste est des couleurs pour pouvoir les choisir rapidement est les ajouter directement aux aspects qui doivent être la couleur demandée.
# Décisions Flexbox pour les layouts complexes:
Mes décisions étaient de vouloir les placer au moins le plus proche possible, si je pouvais, à la location demandé dans le concept du site web.
# Choix de fonctions CSS fluides:
Mes choix pour leurs utilisations étaient qu'ils étaient trop importants à ne pas changer de valeur si jamais la page web change de dimension.
# Défis techniques rencontrés et vos solutions:
PLusieurs de mes défis rencontrés étaient les créations de zones séparées dans une page web et les allignements Flexbox de ces zones. Pour être honête, je n'ai pas vraiment trouver de solution pour ce problème, alors il reste dans mon projet tristement.
